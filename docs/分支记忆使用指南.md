# 分支级记忆隔离使用指南

## 概述

分支级记忆隔离允许每个 git 分支拥有独立的记忆空间，同时支持跨分支读取和引用。

## 快速开始

### 1. 启用分支记忆

```bash
# Linux/macOS
export CLAUDE_CODE_BRANCH_MEMORY=1

# Windows PowerShell
$env:CLAUDE_CODE_BRANCH_MEMORY=1

# Windows CMD
set CLAUDE_CODE_BRANCH_MEMORY=1
```

### 2. 验证启用状态

```bash
# 在 Claude Code 中运行
/memory current-branch
```

输出示例：
```
Current branch: feature/auth
Branch memory enabled: true
Memory path: ~/.claude/projects/my-repo/feature--auth/memory/
```

## 目录结构

启用后，记忆文件按以下结构组织：

```
~/.claude/projects/<repo>/
├── shared/memory/           # 所有分支共享（默认）
│   └── MEMORY.md
├── branches/
│   ├── main/memory/         # main 分支
│   │   ├── MEMORY.md
│   │   └── team/MEMORY.md   # 团队记忆（也按分支隔离）
│   ├── feature-x/memory/    # feature-x 分支
│   │   └── MEMORY.md
│   └── hotfix-123/memory/   # hotfix-123 分支
│       └── MEMORY.md
└── logs/                    # 日志文件（共享）
```

## 命令行用法

### 查看可用分支记忆

```bash
/memory list
```

输出示例：
```
Available branch memories:

  main:
    - MEMORY.md
    - team/MEMORY.md
  feature-x:
    - MEMORY.md
  hotfix-123:
    - MEMORY.md
```

### 从其他分支读取记忆

```bash
# 读取 main 分支的记忆到当前上下文
/memory read-from main

# 读取 feature-x 分支的记忆
/memory read-from feature-x
```

读取的记忆会显示在上下文中：
```
<memory from="main">
## 项目架构设计
- 使用微服务架构
- API 网关采用 Kong
- 数据库使用 PostgreSQL
</memory>

Memory loaded from branch: main
```

### 编辑当前分支记忆

```bash
# 打开记忆编辑器（默认行为）
/memory

# 或使用子命令
/memory edit
```

## 使用场景

### 场景 1：特性分支参考主分支设计

在 `feature` 分支开发时，参考 `main` 分支的架构设计：

```bash
# 切换到特性分支
git checkout -b feature/new-auth

# 在 Claude Code 中
/memory read-from main

# 现在 Claude 了解 main 分支的架构设计
# 可以基于此实现新功能
```

### 场景 2：多分支并行开发

同时开发多个特性，每个特性有独立的记忆：

```bash
# 分支 A：数据库重构
git checkout -b feature/db-refactor
# 创建数据库相关的记忆
# ~/.claude/projects/repo/branches/feature--db-refactor/memory/MEMORY.md

# 分支 B：UI 改进
git checkout -b feature/ui-improvements
# 创建 UI 相关的记忆
# ~/.claude/projects/repo/branches/feature--ui-improvements/memory/MEMORY.md

# 切换回 main 分支
git checkout main
# 使用共享记忆
# ~/.claude/projects/repo/shared/memory/MEMORY.md
```

### 场景 3：热点分支参考主分支

```bash
# 创建热点分支
git checkout -b hotfix/security-patch

# 参考 main 分支的安全规范
/memory read-from main

# Claude 会考虑 main 分支的安全编码规范
```

## @include 跨分支引用

在 MEMORY.md 中直接引用其他分支的记忆：

```markdown
# 当前分支 MEMORY.md

## 基础架构（引用自 main）

@include ~/.claude/projects/my-repo/branches/main/memory/MEMORY.md

## 当前分支特定实现

### 数据库迁移
...
```

**注意**：使用绝对路径引用，分支名中的 `/` 需要替换为 `--`。

## 团队记忆

启用分支记忆后，团队记忆也会自动按分支隔离：

```bash
# 每个分支的团队记忆独立同步
~/.claude/projects/<repo>/
├── branches/main/memory/team/MEMORY.md
├── branches/feature-x/memory/team/MEMORY.md
└── shared/memory/team/MEMORY.md    # 共享团队记忆
```

## 关闭分支记忆

```bash
# 取消环境变量
unset CLAUDE_CODE_BRANCH_MEMORY

# 或使用 0/empty/false
export CLAUDE_CODE_BRANCH_MEMORY=0
```

关闭后，所有分支共享 `shared/memory/` 目录。

## 分支命名规则

分支名会被 sanitize 以适配文件系统：

| 原始分支名 | 目录名 |
|-----------|--------|
| `main` | `main` |
| `feature/auth` | `feature--auth` |
| `hotfix-123` | `hotfix-123` |
| `feat/my-feature` | `feat--my-feature` |
| `WIP 测试` | `WIP-测试` |

规则：
- `/` → `--`
- 其他非法字符 → `-`
- 最大长度：100 字符

## 常见问题

### Q: 启用分支记忆后，之前的记忆会丢失吗？

A: 不会。默认情况下所有记忆存储在 `shared/memory/`。启用分支记忆后：
- 现有记忆保持在 `shared/memory/`
- 新分支记忆存储在 `branches/<branch>/memory/`
- 两个位置的记忆可以同时存在

### Q: 如何在所有分支共享某些记忆？

A: 使用 `shared/memory/` 目录存放共享记忆，或在各分支 MEMORY.md 中使用 `@include` 引用。

### Q: 删除分支后，对应的记忆会自动删除吗？

A: 不会。分支记忆会保留直到手动删除：

```bash
# 删除已合并分支的记忆
rm -rf ~/.claude/projects/<repo>/branches/merged-branch-name/memory/
```

### Q: 性能影响？

A: 分支记忆仅在切换分支时重新计算路径，对日常使用无显著影响。记忆文件缓存机制保持不变。

## 最佳实践

1. **主分支使用共享记忆**：`main`/`master` 分支建议使用 `shared/memory/`
2. **长期分支启用隔离**：超过 1 周的特性分支建议启用分支记忆
3. **定期清理**：合并删除分支后，清理对应的记忆目录
4. **文档化**：在分支 MEMORY.md 中记录分支目标和状态

## 技术细节

- 环境变量：`CLAUDE_CODE_BRANCH_MEMORY=1`
- 路径缓存：基于 `projectRoot + branch` 的 memoization
- Git 检测：使用 `git rev-parse --abbrev-ref HEAD` 获取分支名
- 安全：分支名 sanitize 防止路径遍历攻击
