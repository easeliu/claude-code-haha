# Claude Code Haha 项目完整文档

## 项目概述

**Claude Code Haha** 是基于 Claude Code 泄露源码修复的本地可运行版本，支持接入任意 Anthropic 兼容 API（如 MiniMax、OpenRouter 等）。

原始泄露源码无法直接运行，本仓库修复了启动链路中的多个阻塞问题，使完整的 Ink TUI 交互界面可以在本地工作。

### 核心功能

- **完整的 Ink TUI 交互界面**：与官方 Claude Code 一致的终端用户界面
- **无头模式（--print）**：支持脚本/CI 场景的单次问答
- **MCP 服务器支持**：Model Context Protocol 集成
- **插件系统**：支持扩展功能
- **Skills 系统**：预定义技能执行
- **自定义 API 端点和模型**：支持第三方 API 服务
- **降级 Recovery CLI 模式**：当 TUI 出现问题时的简化交互模式

## 核心模块描述

### 1. 入口层 (`bin/`, `src/entrypoints/`)

#### `bin/claude-haha`
入口 Bash 脚本，负责：
- 检测 `CLAUDE_CODE_FORCE_RECOVERY_CLI` 环境变量
- 选择启动完整 TUI 或降级 CLI 模式
- 设置环境变量并启动 Bun

#### `src/entrypoints/cli.tsx`
CLI 主入口，负责：
- 快速路径检测（--version, --help, 等特殊标志）
- 启动性能分析
- 加载主应用逻辑

#### `src/main.tsx`
应用主逻辑（~80 万行），负责：
- Commander.js 命令注册
- 环境变量和配置加载
- 插件和 Skills 初始化
- MCP 服务管理
- 渲染 TUI 界面

### 2. 界面层 (`src/screens/`, `src/components/`)

#### `src/screens/REPL.tsx`
主交互界面组件，负责：
- 消息渲染和输入处理
- 工具调用显示

- 键盘快捷键处理

## 配置方式

### 环境变量配置（`.env`）

```bash
# API 认证（二选一）
ANTHROPIC_API_KEY=sk-xxx          # 标准 API Key（x-api-key 头）
ANTHROPIC_AUTH_TOKEN=sk-xxx       # Bearer Token（Authorization 头）

# API 端点（可选，默认 Anthropic 官方）
ANTHROPIC_BASE_URL=https://api.minimaxi.com/anthropic

# 模型配置
ANTHROPIC_MODEL=MiniMax-M2.7-highspeed
ANTHROPIC_DEFAULT_SONNET_MODEL=MiniMax-M2.7-highspeed
ANTHROPIC_DEFAULT_HAIKU_MODEL=MiniMax-M2.7-highspeed
ANTHROPIC_DEFAULT_OPUS_MODEL=MiniMax-M2.7-highspeed

# 超时（毫秒）
API_TIMEOUT_MS=3000000

# 禁用遥测和非必要网络请求
DISABLE_TELEMETRY=1
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
```

### 环境变量说明

| 变量 | 必填 | 说明 |
|------|------|------|
| `ANTHROPIC_API_KEY` | 二选一 | API Key，通过 `x-api-key` 头发送 |
| `ANTHROPIC_AUTH_TOKEN` | 二选一 | Auth Token，通过 `Authorization: Bearer` 头发送 |
| `ANTHROPIC_BASE_URL` | 否 | 自定义 API 端点 |
| `ANTHROPIC_MODEL` | 否 | 默认模型 |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | 否 | Sonnet 级别模型映射 |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | 否 | Haiku 级别模型映射 |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | 否 | Opus 级别模型映射 |
| `API_TIMEOUT_MS` | 否 | API 请求超时（毫秒） |
| `DISABLE_TELEMETRY` | 否 | 设为 `1` 禁用遥测 |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | 否 | 设为 `1` 禁用非必要网络请求 |
| `CLAUDE_CODE_FORCE_RECOVERY_CLI` | 否 | 设为 `1` 使用降级 CLI 模式 |

## 运行方法

### 环境要求

- [Bun](https://bun.sh) >= 1.1
- Node.js >= 18

### 安装

```bash
# 克隆仓库后进入目录
cd claude-code-haha

# 安装依赖
npm install
# 或
bun install
```

### 配置

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，填入你的 API Key
```

### 启动

```bash
# 交互 TUI 模式（完整界面）
./bin/claude-haha

# 或
bun run claude-haha

# 无头模式（单次问答）
./bin/claude-haha -p "your prompt here"

# 管道输入
echo "explain this code" | ./bin/claude-haha -p

# 降级模式（简化 CLI）
CLAUDE_CODE_FORCE_RECOVERY_CLI=1 ./bin/claude-haha

# 查看帮助
./bin/claude-haha --help

```

### Windows


前置要求：必须安装 Git for Windows（提供 Git Bash，项目内部 Shell 执行依赖它）。

Windows 下启动脚本 bin/claude-haha 是 bash 脚本，无法在 cmd / PowerShell 中直接运行。请使用以下方式：

## 方式一：PowerShell / cmd 直接调用 Bun（推荐）

# 交互 TUI 模式
bun --env-file=.env ./src/entrypoints/cli.tsx

# 无头模式
bun --env-file=.env ./src/entrypoints/cli.tsx -p "your prompt here"

# 降级 Recovery CLI
bun --env-file=.env ./src/localRecoveryCli.ts

## 方式二：Git Bash 中运行

# 在 Git Bash 终端中，与 macOS/Linux 用法一致
./bin/claude-haha
注意：部分功能（语音输入、Computer Use、Sandbox 隔离等）在 Windows 上不可用，不影响核心 TUI 交互。



### 命令行选项

```
Usage: claude-haha [options] [command]

Options:
  -p, --prompt <text>    直接运行提示并退出
  -v, --version          显示版本号
  -h, --help             显示帮助

Commands:
  mcp                    MCP 服务器管理
  model                  模型选择
  theme                  主题选择
  settings               设置
  help                   帮助
```

## 相对于原始泄露源码的修复

| 问题 | 根因 | 修复 |
|------|------|------|
| TUI 不启动 | 入口脚本把无参数启动路由到了 recovery CLI | 恢复走 `cli.tsx` 完整入口 |
| 启动卡死 | `verify` skill 导入缺失的 `.md` 文件，Bun text loader 无限挂起 | 创建 stub `.md` 文件 |
| `--print` 卡死 | `filePersistence/types.ts` 缺失 | 创建类型桩文件 |
| `--print` 卡死 | `ultraplan/prompt.txt` 缺失 | 创建资源桩文件 |
| **Enter 键无响应** | `modifiers-napi` native 包缺失，`isModifierPressed()` 抛异常 | 加 try-catch 容错 |
| setup 被跳过 | `preload.ts` 自动设置 `LOCAL_RECOVERY=1` 跳过全部初始化 | 移除默认设置 |

## 开发指南

### 添加新工具

1. 在 `src/tools/` 下创建新的工具目录
2. 实现 `Tool` 接口
3. 在 `src/tools.ts` 中注册

### 添加新命令

1. 在 `src/commands/` 下创建命令目录
2. 实现命令逻辑
3. 在 `src/commands.ts` 中注册

### 修改 UI 组件

1. 组件位于 `src/components/`
2. 使用 Ink（React for Terminal）编写
3. 遵循现有组件模式

## 注意事项

1. **API Key 安全**: 不要将 `.env` 文件提交到版本控制
2. **模型兼容性**: 使用兼容 Claude API 的模型可获得最佳效果
3. **终端支持**: 需要支持真彩色和 Unicode 的终端
4. **内存需求**: 建议至少 4GB 可用内存
