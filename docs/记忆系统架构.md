# Claude Code 记忆系统完整架构分析

## 1. 概述

Claude Code 的记忆系统是一个多层次、可扩展的持久化上下文管理机制，允许用户在会话之间保存和复用信息。系统支持从简单的用户级记忆到复杂的团队共享记忆。

### 1.1 核心设计目标

- **持久化**: 跨会话保留上下文和指令
- **层次化**: 不同作用域的记忆（用户、项目、团队）
- **安全性**: 防止凭据泄露和路径遍历攻击
- **可扩展**: 支持自动记忆生成和团队协作

### 1.2 记忆类型（优先级从低到高）

| 类型 | 文件位置 | 作用域 | 是否提交到 Git |
|------|----------|--------|----------------|
| **Managed** | `/etc/claude-code/CLAUDE.md` | 全局（所有用户） | N/A（系统级） |
| **User** | `~/.claude/CLAUDE.md` | 用户全局 | 否 |
| **Project** | `CLAUDE.md`, `.claude/CLAUDE.md`, `.claude/rules/*.md` | 项目级 | 是 |
| **Local** | `CLAUDE.local.md` | 项目级私有 | 否 |
| **AutoMem** | `<memoryBase>/projects/<project>/memory/MEMORY.md` | 自动记忆 | 否 |
| **TeamMem** | `<autoMemPath>/team/` | 团队共享 | 通过同步服务 |

#### 3.3.2 符号链接处理

```typescript
// realpathDeepestExisting: 递归向上查找已存在的祖先
async function realpathDeepestExisting(absolutePath: string): Promise<string> {
  const tail: string[] = []
  let current = absolutePath
  
  for (let parent = dirname(current); current !== parent; ) {
    try {
      const realCurrent = await realpath(current)
      // 重新连接不存在的尾部
      return tail.length === 0 ? realCurrent : join(realCurrent, ...tail.reverse())
    } catch (e) {
      if (e.code === 'ENOENT') {
        // 检查是否为悬空符号链接
        const st = await lstat(current)
        if (st.isSymbolicLink()) {
          throw new PathTraversalError(`Dangling symlink: ${current}`)
        }
        tail.push(current.slice(parent.length + sep.length))
        current = parent
      }
    }
  }
}
```

### 3.4 团队记忆同步 (`teamMemorySync/`)

#### 3.4.1 同步状态管理

  lastKnownChecksum: string | null  // 服务端 ETag

#### 3.4.2 拉取流程 (Pull)

```typescript
async function pullTeamMemory(state: SyncState): Promise<{
  success: boolean
  filesWritten: number
  entryCount: number
}> {
  // 1. 带条件请求 (If-None-Match) 获取服务端数据
  const result = await fetchTeamMemory(state, repoSlug, etag)
  
  // 2. 处理响应
  if (result.status === 304) return { notModified: true }
  if (result.status === 404) return { isEmpty: true }
  
  // 3. 更新 serverChecksums
  state.serverChecksums.clear()
  for (const [key, hash] of Object.entries(responseChecksums)) {
    state.serverChecksums.set(key, hash)
  }
  
  // 4. 跳过内容相同的文件 (避免更新 mtime)
  const filesWritten = await writeRemoteEntriesToLocal(entries)
}
```

#### 3.4.3 推送流程 (Push)

```typescript
async function pushTeamMemory(state: SyncState): Promise<{
  success: boolean
  filesUploaded: number
  conflict?: boolean
}> {
  // 1. 读取本地文件（扫描凭据）
  const { entries, skippedSecrets } = await readLocalTeamMemory()
  
  // 2. 计算增量（仅哈希不同的键）
  const delta = {}
  for (const [key, content] of Object.entries(entries)) {
    if (state.serverChecksums.get(key) !== hashContent(content)) {
      delta[key] = content
    }
  }
  
  // 3. 分批推送（避免超过网关限制）
  const batches = batchDeltaByBytes(delta)
  for (const batch of batches) {
    const result = await uploadTeamMemory(batch, ifMatchChecksum)
    if (result.conflict) {
      // 412 冲突处理：获取服务端哈希，重新计算增量
      const probe = await fetchTeamMemoryHashes()
      state.serverChecksums = probe.entryChecksums
    }
  }
}
```

// 解析 frontmatter paths
function parseFrontmatterPaths(rawContent: string): {
  content: string
  paths?: string[]
} {
  const { frontmatter, content } = parseFrontmatter(rawContent)
  
  // 过滤掉纯 ** 模式（匹配所有）
  const patterns = frontmatter.paths
    ?.map(p => p.endsWith('/**') ? p.slice(0, -3) : p)
    .filter(p => p.length > 0 && p !== '**')
  
  return patterns?.length ? { content, paths: patterns } : { content }
}

// 匹配目标路径
function processConditionedMdRules(...): Promise<MemoryFileInfo[]> {
  const files = await processMdRules({ conditionalRule: true })
  return files.filter(file => {
    const ig = ignore().add(file.globs)
    return ig.ignores(relativePath)
  })
}

│  └──────────────┘                  └──────────────┘         │

```

### 8.2 API 端点

```
GET  /api/claude_code/team_memory?repo={owner/repo}&branch={branch}
     → TeamMemoryData (包含 entryChecksums)
     branch 参数可选，默认使用当前分支

GET  /api/claude_code/team_memory?repo={owner/repo}&branch={branch}&view=hashes
     → 仅 metadata + checksums（用于冲突解决）

PUT  /api/claude_code/team_memory?repo={owner/repo}&branch={branch}
     → 上传条目（upsert 语义）
```

**分支参数处理**：
- 如果未指定 branch 参数，服务器默认使用 `main` 分支
- 客户端通过 `CLAUDE_CODE_BRANCH_MEMORY=1` 启用时自动附加当前分支参数
- 团队记忆的分支隔离是可选的，不影响现有的共享团队记忆工作流

### 8.3 冲突解决

```
1. 本地编辑 → 计算增量 delta
2. PUT with If-Match: <etag>
3. 收到 412 Precondition Failed
4. GET ?view=hashes 获取最新服务端 checksums
5. 重新计算 delta（排除服务端已提交的相同内容）
6. 重试 PUT
```

├── branches/
│   ├── main/memory/         # main 分支的记忆
│   │   └── MEMORY.md
│   ├── feature-x/memory/    # feature-x 分支的记忆
│   │   └── MEMORY.md
│   └── hotfix-123/memory/   # hotfix-123 分支的记忆
│       └── MEMORY.md
└── team/                    # 团队记忆（也按分支隔离）
    └── MEMORY.md
```

### 12.3 分支名称处理

- 分支名中的 `/` 会被转换为 `--`（如 `feature/auth-test` → `feature--auth-test`）
- 其他非法字符会被替换为 `-`
- 最大长度限制为 100 字符

### 12.4 跨分支读取

```bash
# 查看可用分支记忆
/memory list

# 查看当前分支信息
/memory current-branch

# 从其他分支读取记忆到当前上下文
/memory read-from feature-x
/memory read-from main
```

### 12.5 @include 跨分支引用

在 MEMORY.md 中使用绝对路径引用其他分支的记忆：

```markdown
# 当前分支的 MEMORY.md

# 引用 main 分支的基础设计
@include ~/.claude/projects/<repo>/branches/main/memory/MEMORY.md

# 当前分支特定的实现
## 实现细节
...
```

### 12.6 团队记忆同步

团队记忆也会按分支隔离同步：

```
GET /api/claude_code/team_memory?repo=owner/repo&branch=main
GET /api/claude_code/team_memory?repo=owner/repo&branch=feature-x
```

同步状态独立维护，每个分支有自己的 checksums。

## 13. 总结

Claude Code 的记忆系统是一个复杂而精密的多层架构：

1. **分层设计**: 6 种记忆类型，从全局到项目到团队，满足不同作用域需求
2. **安全优先**: 三级路径验证、凭据扫描、符号链接防护
3. **增量同步**: 基于哈希的增量推送、冲突解决、防抖优化
4. **灵活扩展**: @include 指令、条件规则、Hooks 系统
5. **性能优化**: 多级缓存、懒加载、文件监听优化

这套系统使得 Claude 能够跨会话理解项目上下文、用户偏好和团队规范，同时保持安全性和性能。
